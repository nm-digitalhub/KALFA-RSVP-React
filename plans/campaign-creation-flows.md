# שני תהליכי יצירת-קמפיין — מסמך הכרעה

**מטרה:** להחליט כיצד נוצר הקמפיין היחיד של אירוע, לאחר הסרת כפתור "קמפיין חדש" ומעבר לקמפיין-אחד-לאירוע.
**קהל:** בעל-המוצר.
**מחוץ להיקף:** מודל החיוב ומחזור-החיים — **זהים לחלוטין בשתי האופציות** ואינם משתנים (per-reached-contact, J5 hold, תקרה, authorized set, charge).

---

## מה משותף לשתי האופציות

בשתיהן, ללא הבדל:

- **אין יותר כפתור "קמפיין חדש"** ואין דף `/campaign/new` ואין טופס בחירה.
- **קמפיין אחד לאירוע** — נאכף ברמת-DB (`UNIQUE(event_id)` חלקי).
- **template קנוני אחד** — נפתר אוטומטית בצד-שרת; המשתמש לא בוחר template ולא מזין תאריכים (נגזרים מתאריך-האירוע).
- **מחזור-החיים זהה:** `pending_approval` → חתימת-הסכם → `approved` → תפיסת-מסגרת J5 → `active` → סגירה → גמר-חשבון.
- **החיוב זהה:** ללא נגיעה.

**ההבדל היחיד בין האופציות:** מתי וכיצד נוצרת שורת-הקמפיין, וכמה "יצירה" המשתמש מודע לה.

---

## אופציה 1 — לחיצת-הקמה אחת (Lazy מפורש)

הקמפיין נוצר ברגע שהמשתמש לוחץ פעם אחת על "הקמת קמפיין אישורים".

### זרימה

```
עמוד האירוע
  └─ מקטע "קמפיין"  (אין עדיין קמפיין)
        ├─ סיכום קצר: מחיר לאיש-קשר, תקרה משוערת, חלון-תאריכים
        └─ [ הקמת קמפיין אישורים ]   ← לחיצה אחת
                 │
                 ▼
        setupCampaignAction(eventId)
          • פותר template קנוני + גוזר תאריכים
          • יוצר שורה  status = pending_approval
                 │
                 ▼
        /campaign/[id]/approve   →   חתימה   →   תשלום(J5)   →   הפעלה ...
```

### מה המשתמש רואה
1. נכנס לאירוע → מקטע "קמפיין" עם הסבר קצר וכפתור-הקמה אחד.
2. לוחץ → עובר ישר למסך החתימה.
3. מאותו רגע המקטע מציג מצב + פעולות (אין יותר כפתור-הקמה).

### יתרונות
- **שקיפות:** המשתמש יודע במפורש שהוא "הקים קמפיין" — רגע-החלטה ברור.
- **אין שורות-רפאים:** קמפיין נוצר רק כשבאמת רוצים בו; אירועים שלא מפעילים קמפיין נשארים נקיים ב-DB.
- **מימוש פשוט ובטוח:** היצירה היא פעולת-שרת אחת מפורשת; קל לבדוק.

### חסרונות
- צעד-לחיצה אחד נוסף לפני החתימה.

### טיפול-קצוות
- **0 אורחים:** הכפתור מציג "הוסיפו אורחים עם טלפון לפני הקמת הקמפיין" (לא נוצרת שורה ריקה).
- **שתי לשוניות / לחיצה כפולה:** `UNIQUE(event_id)` + יצירה אידמפוטנטית → הלחיצה השנייה נוחתת על אותו קמפיין.

---

## אופציה 2 — אוטומטי לחלוטין (ללא לחיצת-יצירה)

אין רגע-יצירה נפרד. מקטע הקמפיין פשוט קיים בעמוד האירוע, והפעולה הראשונה של המשתמש היא **החתימה**. שורת-הקמפיין מסופקת ברקע.

### זרימה

```
עמוד האירוע
  └─ מקטע "קמפיין"  (קיים תמיד)
        ├─ סיכום: מחיר לאיש-קשר, תקרה משוערת, חלון-תאריכים
        └─ [ לאישור וחתימה ]            ← הפעולה הראשונה היא כבר החתימה
                 │
                 ▼
        השורה (status = pending_approval) מסופקת אוטומטית ברקע
        בנקודת-הכניסה למסך האישור
                 │
                 ▼
        חתימה   →   תשלום(J5)   →   הפעלה ...
```

תת-החלטת-מימוש בתוך אופציה זו: *מתי* בדיוק נוצרת השורה —
- **2א – ביצירת השורה בכניסה למקטע/למסך האישור** (פשוט, אך עלול ליצור שורות `pending_approval` למי שרק הציץ).
- **2ב – דחיית היצירה לרגע החתימה** (נקי מ"שורות-רפאים", אך מורכב יותר: מסך-האישור צריך לפעול לפני שקיימת שורה).

### מה המשתמש רואה
1. נכנס לאירוע → מקטע "קמפיין" עם סיכום וכפתור "לאישור וחתימה".
2. לוחץ → ישר לחתימה (אין צעד-הקמה מודע).

### יתרונות
- **מסע קצר יותר:** צעד אחד פחות; הקמפיין מרגיש "חלק מהאירוע" מטבעו.

### חסרונות
- **פחות שקיפות:** אין רגע-החלטה מפורש ("מתי התחלתי קמפיין?").
- **סיכון שורות-רפאים** (ב-2א) או **מורכבות-מימוש** (ב-2ב).
- קשה יותר לבדוק (היצירה מוטמעת ב-render/בחתימה ולא בפעולה אחת).

### טיפול-קצוות
- **0 אורחים:** המקטע מציג "הוסיפו אורחים עם טלפון" ומשבית את "לאישור וחתימה" עד שיש איש-קשר.
- **שתי לשוניות:** כמו באופציה 1 — `UNIQUE(event_id)` + יצירה אידמפוטנטית.

## השוואה צד-לצד

| ממד | אופציה 1 — לחיצת-הקמה | אופציה 2 — אוטומטי |
|---|---|---|
| צעדים עד החתימה | הקמה → חתימה | חתימה |
| שקיפות "התחלתי קמפיין" | גבוהה (מפורש) | נמוכה (סמוי) |
| שורות-רפאים ב-DB | אין | אפשרי (2א) / נמנע במורכבות (2ב) |
| מורכבות מימוש | נמוכה | בינונית–גבוהה |
| קלות בדיקה | גבוהה (פעולה אחת) | נמוכה יותר |
| הסרת כפתור "קמפיין חדש" | ✓ | ✓ |
| אחד-לאירוע + template קנוני | ✓ | ✓ |
| שינוי חיוב/מחזור-חיים | אין | אין |

## המלצה

**אופציה 1 (לחיצת-הקמה אחת).** היא מסירה את כפתור "קמפיין חדש" ומשיגה קמפיין-אחד-לאירוע בדיוק כמו אופציה 2, אבל בלי שורות-רפאים, עם רגע-החלטה שקוף למשתמש, ועם מימוש פשוט ובטוח לבדיקה. הצעד-הנוסף היחיד (לחיצה אחת) זניח מול היתרונות.

אופציה 2 עדיפה רק אם המטרה היא שהקמפיין יהיה בלתי-נראה לחלוטין כישות נפרדת — ואז כדאי דווקא 2ב (דחיית-יצירה) כדי להימנע משורות-רפאים, במחיר מורכבות.

**להכרעתך:** אופציה 1, או אופציה 2 (ואם 2 — 2א או 2ב)?
