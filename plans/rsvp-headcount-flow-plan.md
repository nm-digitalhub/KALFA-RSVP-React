# זרימת ספירת-ראשים אחרי אישור הגעה + ייחוד טלפון

**סטטוס:** תוכנית — ממתין לאישור. דרישת בעלים 2026-07-05:
"טלפון אחד בלי כפילויות; מי שאישר הגעה מקבל הודעת המשך וצריך להשיב מספר
1–10; אם השיב 0 — ההודעה נשלחת שוב; אם לא ענה — 0 דיפולט במערכת."

## A. ייחוד מספר טלפון (ברמת המוזמנים)

היום כפילות טלפון בין מוזמנים היא פיצ'ר (משפחה על מכשיר אחד; contacts כבר
ייחודי פר טלפון). ההחלטה החדשה: **שורת מוזמן אחת לכל טלפון**.

1. מיגרציה: `create unique index guests_event_phone_key on guests(event_id,
   normalized-phone) where phone is not null` — על ביטוי טלפון מנורמל
   (normalizePhone), לא הטקסט הגולמי.
2. ייבוא: כפילות בתוך הקובץ או מול קיימים → שגיאת-שורה בעברית ("מספר כבר
   קיים אצל ‹שם›"), השאר ממשיכות (partial success כרגיל).
3. טופס מוזמן יחיד: אותה בדיקה עם הודעה ידידותית.
4. preflight על ה־DB החי לפני push: אין כפילויות קיימות (יש היום? לבדוק).

## B. הודעת "כמה תגיעו?" (1–10)

**טריגר:** עיבוד לחיצת "מגיע/ה" ב־webhook (אחרי submitRsvp המצליח) —
בתוך חלון 24ש' ⇒ הודעת session חופשית, בלי תבנית ובלי עלות MARKETING:
"תודה! כמה תגיעו בסך הכול? השיבו במספר (1–10)".

**מצב:** עמודות חדשות על `guests`: `headcount_requested_at`,
`headcount_answered_at`, `confirmed_headcount int not null default 0`
(**0 = "טרם נענה" — הדיפולט שביקשת**; הדוחות מציגים אותו ככזה).

**קליטת תשובה:** ב־webhook-processing, הודעת טקסט נכנסת מאיש קשר עם בקשה
פתוחה (`requested_at` מלא, `answered_at` ריק):
- ‏1–10 → שמירה + אישור קצר ("נרשם! מחכים לכם 🎉") + `answered_at`.
- ‏**0 → שליחה חוזרת של השאלה** (עם מונה נסיונות, תקרה 3 — הגנה מלולאה).
- טקסט אחר → מתעלמים (נשאר 0; לא מציקים).
- דה-דופ Meta retries דרך `fresh` הקיים; אין רישום חיוב נוסף (UNIQUE קיים).

**התאמה ל־submit_rsvp:** לא נוגעים בו (דוחה 0 בכוונה — attending=מבוגר 1
כברירת placeholder). הספירה האמיתית חיה ב־`confirmed_headcount`; מסכי
הדוחות/מוזמנים יציגו אותה לצד סטטוס האישור.

## סדר וגייטים
1. מיגרציה (ייחוד + עמודות) + preflight ⇒ אישור push.
2. קוד: ייבוא/טופס + webhook branch + שליחת session + בדיקות (כולל 0-לולאה,
   תקרת נסיונות, טקסט זבל, retry של Meta).
3. פריסה + בדיקת אמת על מספר בדיקה.
