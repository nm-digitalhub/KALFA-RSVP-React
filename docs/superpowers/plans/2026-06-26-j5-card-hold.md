# J5 Card-Hold at Campaign Approval (Route A) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** After signing the agreement, the event owner enters a credit card and KALFA places a SUMIT J5 authorization hold up to the campaign's charge ceiling (the actual charge happens later, at campaign close).

**Architecture:** Reuse the proven SUMIT `payments.js` card-form pattern (jQuery → payments.js → `BindFormSubmit` → `og-token` → native POST). The card form lives on the existing campaign `payment` page and POSTs the single-use token to a new Route Handler `POST /api/campaigns/[id]/authorize`. The handler verifies ownership, atomically locks the campaign's `capture_status` (idempotency), calls a new `authorizeHoldSumit` adapter (`AutoCapture:false`, `AuthorizeAmount=ceiling`, `PreventDocumentCreation:true`), and persists `auth_number` + `auth_amount` + `authorized_at` + `auth_expires_at` + `card_token_ref` + `capture_status='authorized'` on the campaign. The campaign status stays `approved`; a failed hold leaves `capture_status='hold_failed'` and is retryable — it never un-signs the agreement.

**Tech Stack:** Next.js 16 App Router (Route Handlers, Server Components), Supabase service-role client for the hold write, SUMIT REST `billing/payments/charge/` (J5), Zod 4, Vitest. No new npm packages.

## Global Constraints

- **Config-gated / fail-closed:** the card form and the hold call run ONLY when `payments_enabled = true` AND a new `campaign_holds_enabled = true` AND `getSumitServerConfig()` returns config. Any missing → no SUMIT call, render the informational state. Default OFF.
- **No real charge in tests/build:** `npm run test` and `next build` must NEVER reach live SUMIT — the adapter is unit-tested with a mocked `fetch`.
- **Money is server-derived:** the hold amount = `campaign.max_charge_ceiling` read server-side. NEVER trust an amount from the browser.
- **Amounts cross JS as strings** (Postgres `numeric`), `parseFloat` only at the SUMIT boundary (mirror `charge.ts`/`raw-charge.ts`). No decimal library.
- **PII:** never store `CitizenID`; never log the SUMIT raw response, `auth_number`, `card_token_ref`, or card data.
- **Idempotency:** a retry must not stack holds — guard on `capture_status` before charging.
- **Migrations hit the LIVE Supabase** ([[supabase-live-schema]]): write the `.sql`, introspect first via `scripts/sb-query.mjs`, and DO NOT apply without explicit user approval.
- **Build discipline:** one `npm run build` at the end, then `pm2 restart kalfa-beta` immediately; tell the user to hard-refresh (Server Action / chunk IDs change).
- **Verified SUMIT facts** [[sumit-charge-verified-behavior]]: J5 needs `Item.Name` (not just Description), `AutoCapture:false`, `PreventDocumentCreation:true`; response returns `Payment.AuthNumber` + reusable `CreditCard_Token`. ⚠️ The exact success/error discriminator (`Status` enum 0/1/2 vs `Status.IsError`) MUST be confirmed against a live POC response before go-live (Task 8).

- [ ] **Step 3: Implement `authorizeHoldSumit`** (`src/lib/sumit/authorize.ts`)

```ts
import 'server-only';

import { SumitDeclinedError, SumitNetworkError } from './charge';

const SUMIT_CHARGE_URL = 'https://api.sumit.co.il/billing/payments/charge/';

export interface SumitAuthorizeParams {
  companyId: number;
  apiKey: string;
  ogToken: string;
  ceiling: string;   // Postgres numeric as string — the J5 hold amount
  vatRate: string;
  authRef: string;   // Customer.ExternalIdentifier — reconciliation anchor
  customerEmail: string;
}

export interface SumitAuthorizeResult {
  authNumber: string;
  cardToken: string | null; // reusable CreditCard_Token for the final capture
}

// J5 authorization HOLD (no capture, no document). Mirrors charge.ts error
// semantics: only Status.IsError/Status===1 in a 2xx body is a definite decline;
// anything else ambiguous → network error (caller marks hold_review, never
// silently "authorized"). NB: confirm the live success discriminator in Task 8.
export async function authorizeHoldSumit(p: SumitAuthorizeParams): Promise<SumitAuthorizeResult> {
  const body = {
    Credentials: { CompanyID: p.companyId, APIKey: p.apiKey },
    Customer: {
      EmailAddress: p.customerEmail || undefined,
      ExternalIdentifier: p.authRef,
    },
    VATIncluded: true,
    VATRate: parseFloat(p.vatRate),
    Items: [{
      Quantity: 1,
      UnitPrice: parseFloat(p.ceiling),
      Item: { Name: 'KALFA — תפיסת מסגרת לקמפיין' },
      Description: 'KALFA — תפיסת מסגרת לקמפיין',
    }],
    SingleUseToken: p.ogToken,
    AutoCapture: false,                 // J5 = authorize/hold only
    AuthorizeAmount: parseFloat(p.ceiling),
    PreventDocumentCreation: true,      // hold → no Order document
    SendDocumentByEmail: false,
    DraftDocument: false,
  };

  let res: Response;
  try {
    res = await fetch(SUMIT_CHARGE_URL, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(body),
    });
  } catch {
    throw new SumitNetworkError('שגיאת תקשורת עם מערכת התשלום');
  }
  if (!res.ok) throw new SumitNetworkError('לא התקבל אישור חד משמעי ממערכת התשלום');

  type Resp = {
    Status?: number | { IsError?: boolean };
    Data?: { Payment?: { AuthNumber?: string | null }; CreditCard_Token?: string | null };
  };
  let json: Resp;
  try {
    json = (await res.json()) as Resp;
  } catch {
    throw new SumitNetworkError('תגובה לא תקינה ממערכת התשלום');
  }

  // Definite decline: numeric Status===1 OR { IsError:true } (handle both shapes
  // until the live shape is pinned in Task 8).
  const isError =
    json.Status === 1 ||
    (typeof json.Status === 'object' && json.Status?.IsError === true);
  if (isError) throw new SumitDeclinedError();

  const authNumber = json.Data?.Payment?.AuthNumber;
  if (!authNumber) throw new SumitNetworkError('אישור התפיסה לא התקבל ממערכת');
  return { authNumber, cardToken: json.Data?.CreditCard_Token ?? null };
}
```
