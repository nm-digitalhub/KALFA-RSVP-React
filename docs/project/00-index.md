# KALFA Event Magic — תיעוד מלא של הפרויקט

מסמך זה הוא האינדקס של סט התיעוד המלא של פרויקט KALFA Event Magic — פלטפורמת RSVP מסוג B2C לניהול אירועים פרטיים, מוזמנים, הזמנות ותשובות (אישורי הגעה), בעברית וב‑RTL כברירת מחדל.

תאריך הפקה: 2026‑07‑02. התיעוד הופק מקריאה ישירה של קוד המקור, המיגרציות והקונפיגורציה בריפו `/var/www/vhosts/kalfa.me/beta` (branch: `main`), ולא מזיכרון או ממסמכי תכנון בלבד. פריטים שקיימים רק בתכנון מסומנים במפורש "מתוכנן, טרם מומש".

## עקרונות רוחביים (תקציר)

- **אבטחה בצד השרת**: כל הרשאה ובעלות נאכפות בשרת (Server Actions / Route Handlers); RLS הוא שכבת הגנה נוספת, לא תחליף.
- **בעלות על נתונים**: אירועים שייכים לבעליהם; כל רשומה נגזרת (מוזמנים, קמפיינים, דוחות, חיובים) נבדקת דרך גבול הבעלות של האירוע.
- **פרטיות**: שמות, טלפונים, תשובות RSVP והיסטוריית הודעות הם מידע אישי; אין רישום PII/סודות בלוגים; תשובות ספקי תשלום עוברות redaction בגישת allow‑list (fail‑closed).
- **נתונים עסקיים מנוהלי‑DB**: מחירים, ערוצים, מסלולים ומדיניות נקראים מנתוני admin במסד — לעולם לא hardcoded בקוד או ב‑UI.
- **עברית ו‑RTL תחילה**: הממשק עברי‑ראשון עם תמיכה עתידית באנגלית וצרפתית.
- **אימות לפני סיום**: `npm run lint`, `npx tsc --noEmit`, vitest ו‑build הם תנאי Definition of Done.

## הערות שימוש (NotebookLM)

- כל מסמך עומד בפני עצמו וכולל הפניות לקבצי מקור בפורמט `path:line`.
- מונחים טכניים, שמות טבלאות, פונקציות ונתיבים נשמרים באנגלית בכוונה — כך מופיעים בקוד.
- אם יש סתירה בין מסמך תכנון (`plans/`) לבין הקוד — הקוד הוא מקור האמת, והמסמכים כאן משקפים את הקוד נכון לתאריך ההפקה.
